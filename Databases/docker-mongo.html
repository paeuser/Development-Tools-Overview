Instructions for Using MongoDB with Docker

How to Check if the MongoDB Docker Image is Already Installed

  1. Open PowerShell or Command Prompt.
  2. Run the following command:
    docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.ID}}" | findstr mongodb-community-server
  3. If you see a result, the image is already installed. If not, proceed to pull the image as described below.

How to Pull the MongoDB Docker Image

  1. Open PowerShell or Command Prompt.
  2. Run the following command:
    docker pull mongodb/mongodb-community-server
  3. This will download the latest MongoDB Community Server image from Docker Hub.

How to Run MongoDB Docker Image

  1. After the image is downloaded, you can run it using the following command:
    docker run --name mongodb -d mongodb/mongodb-community-server
  2. This command will run the MongoDB image in a new container named "mongodb".

How to Run MongoDB for Local Development

  1. To ensure your application can connect using mongodb://localhost:27017/?ssl=false, run the following command:
    docker run --name mongodb -p 27017:27017 -d mongodb/mongodb-community-server
  2. This maps port 27017 on your computer to port 27017 in the container, so localhost:27017 works for your app.
  3. If you already have a container named mongodb, remove it first with:
    docker rm mongodb

How to Check if MongoDB is Running

  1. Open PowerShell or Command Prompt.
  2. Run the following command:
    docker ps --filter "name=mongodb"
  3. If you see a line with the container details, MongoDB is running.
    If nothing is returned, the container is not running.
  4. To check all containers (including stopped ones), run:
    docker ps -a --filter "name=mongodb"

How to Connect to MongoDB

  1. To connect to the MongoDB instance running in the container, use the following command:
    docker exec -it mongodb mongo
  2. This command will open the MongoDB shell connected to your MongoDB instance.
