What is LINQ? LINQ (Language Integrated Query) is a .NET framework component that adds native data querying capabilities to .NET languages using a syntax similar to SQL.
What is deferred execution in LINQ? Deferred execution means that a LINQ query is not executed until its results are enumerated, allowing for optimized query execution.
What is the difference between SQL and LINQ? SQL is a query language for relational databases, while LINQ is a .NET framework feature that allows querying of various data sources, including databases, collections, and XML, using a unified syntax.
What is a lambda expression in LINQ? A lambda expression is an anonymous function used to create delegates or expression tree types, often used in LINQ queries for concise syntax.
What are LINQ query operators? LINQ query operators are methods that operate on sequences and allow querying, filtering, and transforming data. Examples include Select, Where, and OrderBy.
What is LINQ to SQL? LINQ to SQL is a component of LINQ that provides a runtime infrastructure for managing relational data as objects without losing the ability to query.
What is LINQ to XML? LINQ to XML is a LINQ provider that allows querying and manipulating XML documents using LINQ syntax.
What is the purpose of the GroupBy operator in LINQ? The GroupBy operator in LINQ is used to group elements of a sequence based on a specified key selector function.
What is the difference between LINQ and Entity Framework? LINQ is a query syntax in .NET for querying data, while Entity Framework is an ORM (Object-Relational Mapper) that uses LINQ to query databases.
What is the Select operator in LINQ? The Select operator is used to project each element of a sequence into a new form, often transforming the data.
What is LINQ to Objects? LINQ to Objects allows querying in-memory collections like arrays, lists, and dictionaries using LINQ syntax.
What is the purpose of the Where operator in LINQ? The Where operator is used to filter a sequence of values based on a predicate function.
What is the difference between First and FirstOrDefault in LINQ? First throws an exception if no elements are found, while FirstOrDefault returns a default value instead.
