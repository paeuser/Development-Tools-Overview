Git Notes

  • git rebase: Moves or combines a sequence of commits to a new base commit. Useful for updating a feature branch with the latest changes from main, or for cleaning up commit history. Rewrites commit history to make it linear.
  • git merge: Combines changes from one branch into another, creating a new merge commit. Preserves the full branch history.
  • git push: Uploads local commits to a remote repository.
  • git pull: Fetches and integrates changes from a remote repository into your current branch.
  • git commit --amend: Modifies the most recent commit.
  • git reset: Moves the current branch to a specified commit, optionally modifying the working directory and staging area.
  • git log: Shows the commit history for the current branch.
  • git status: Displays the state of the working directory and staging area.

Note: If you accidentally commit sensitive information (like API tokens or passwords) to your repository, GitHub will block your push to protect your project. In these cases, you must remove the secret from your files and also from your git history. This is why we use git rebase (or similar history-rewriting commands) to edit or remove problematic commits before pushing again. Rebasing allows you to rewrite your commit history, ensuring that secrets are completely removed from all previous commits, not just the latest one.
