.NET Framework and .NET Core Interview Questions & Answers

  1. What is the .NET Framework?
    A software development platform developed by Microsoft for building and running Windows applications. It includes the CLR, class libraries, and supports multiple languages.
  2. What is .NET Core?
    A cross-platform, open-source version of .NET for building applications that run on Windows, Linux, and macOS. Now part of .NET 5+ as ".NET".
  3. What is the CLR?
    The Common Language Runtime is the virtual machine component of .NET that manages memory, code execution, security, and more.
  4. What is the difference between .NET Framework and .NET Core?
    .NET Framework is Windows-only and mature; .NET Core is cross-platform, lightweight, and modern. .NET 5+ unifies both.
  5. What is the CTS?
    Common Type System defines how types are declared, used, and managed in the runtime, ensuring language interoperability.
  6. What is the CLS?
    Common Language Specification is a set of rules for .NET languages to ensure interoperability.
  7. What is the GAC?
    Global Assembly Cache is a machine-wide code cache for storing shared .NET assemblies.
  8. What is an assembly?
    A compiled code library used for deployment, versioning, and security. It can be a DLL or EXE.
  9. What is NuGet?
    NuGet is the package manager for .NET, used to share and consume reusable code.
  10. What is dependency injection?
    A design pattern for achieving Inversion of Control, where dependencies are provided to a class rather than created by it. Built-in in ASP.NET Core.
  11. What is ASP.NET Core?
    A cross-platform, high-performance framework for building modern web, cloud, and API applications.
  12. What is Entity Framework?
    An ORM (Object-Relational Mapper) for .NET, enabling developers to work with databases using .NET objects.
  13. What is middleware in ASP.NET Core?
    Software components that are assembled into an application pipeline to handle requests and responses.
  14. What is Kestrel?
    A cross-platform web server for ASP.NET Core applications.
  15. What is the difference between Task and Thread?
    A Thread is a lower-level OS construct; a Task is a higher-level abstraction for asynchronous programming in .NET.
  16. What is async/await?
    Keywords in C# for asynchronous programming, allowing non-blocking code execution.
  17. What is a delegate?
    A type that represents references to methods with a specific parameter list and return type.
  18. What is a lambda expression?
    An anonymous function that can contain expressions and statements, used to create delegates or expression tree types.
  19. What is LINQ?
    Language Integrated Query, a set of features for querying data from different sources in a consistent way.
  20. What is boxing and unboxing?
    Boxing is converting a value type to object; unboxing is extracting the value type from the object.
  21. What is garbage collection?
    Automatic memory management that frees unused objects to reclaim memory.
  22. What is a value type vs reference type?
    Value types store data directly; reference types store references to the data's memory address.
  23. What is a nullable type?
    A value type that can also represent null, e.g., int? in C#.
  24. What is a generic?
    A feature that allows classes and methods to operate on types specified by the programmer at compile time.
  25. What is reflection?
    The ability of a program to inspect and modify its own structure and behavior at runtime.
  26. What is a RESTful API?
    An API that follows REST principles, using HTTP methods for CRUD operations and stateless communication.
  27. What is JWT?
    JSON Web Token, a compact, URL-safe means of representing claims to be transferred between two parties, often used for authentication.
  28. What is SignalR?
    A library for ASP.NET that enables real-time web functionality, such as live chat or notifications.
  29. What is a microservice?
    An architectural style that structures an application as a collection of loosely coupled services.
  30. What is the difference between Web API and MVC?
    Web API is for building HTTP services; MVC is for building web applications with views and controllers.
  31. What is dependency inversion principle?
    A SOLID principle stating that high-level modules should not depend on low-level modules, but both should depend on abstractions.
  32. What is the difference between abstract class and interface?
    Abstract classes can have implementation; interfaces cannot (prior to C# 8). A class can implement multiple interfaces but inherit only one abstract class.
  33. What is a sealed class?
    A class that cannot be inherited.
  34. What is a static class?
    A class that cannot be instantiated and contains only static members.
  35. What is a partial class?
    A class whose definition can be split across multiple files.
  36. What is a singleton pattern?
    A design pattern that restricts a class to a single instance and provides a global point of access to it.
  37. What is the difference between IEnumerable and IQueryable?
    IEnumerable executes queries in memory; IQueryable can translate queries to the data source (e.g., SQL) for remote execution.
  38. What is middleware?
    Software that handles requests and responses in the ASP.NET Core pipeline.
  39. What is the Startup.cs file?
    A configuration file in ASP.NET Core that defines services and the app's request pipeline.
  40. What is appsettings.json?
    A configuration file in ASP.NET Core for storing settings like connection strings and app configuration.
  41. What is the difference between synchronous and asynchronous programming?
    Synchronous code blocks the thread until completion; asynchronous code allows other operations to run while waiting.
  42. What is a memory leak?
    A situation where memory that is no longer needed is not released, leading to increased memory usage.
  43. What is a deadlock?
    A situation where two or more threads are blocked forever, each waiting for the other to release a resource.
  44. What is a race condition?
    A situation where the outcome depends on the sequence or timing of uncontrollable events, often in multithreaded code.
  45. What is the difference between public, private, protected, and internal access modifiers?
    public: accessible everywhere; private: only within the class; protected: within the class and derived classes; internal: within the same assembly.
  46. What is a constructor?
    A special method called when an object is instantiated, used to initialize the object.
  47. What is method overloading and overriding?
    Overloading: same method name, different parameters; overriding: derived class provides a new implementation for a base class method.
  48. What is polymorphism?
    The ability of different classes to be treated as instances of the same base class, typically via interfaces or inheritance.
  49. What is encapsulation?
    The bundling of data and methods that operate on that data within a class, restricting direct access to some of the object's components.
  50. What is inheritance?
    A mechanism where a new class derives from an existing class, inheriting its members.
  51. What is a namespace?
    A container for classes and other types, used to organize code and prevent naming conflicts.
  52. What is a try-catch block?
    A construct for handling exceptions in .NET code.
