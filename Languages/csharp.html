abstract class vs interface? An abstract class can have implementations for some of its members, but an interface can only have declarations.
access modifiers? Access modifiers define the scope and visibility of a class, method, or variable. Examples include public, private, protected, and internal.
async and await? The "async" and "await" keywords are used to write asynchronous code, allowing tasks to run without blocking the main thread.
boxing and unboxing? Boxing is the process of converting a value type to an object type, while unboxing is the reverse process.
constructor? A constructor is a special method in a class that is called when an object of the class is created. It is used to initialize the object.
const vs readonly? "const" is a compile-time constant, while "readonly" is a runtime constant that can only be assigned during declaration or in a constructor.
delegate? A delegate is a type that represents references to methods with a specific parameter list and return type.
dependency injection? Dependency injection is a design pattern used to achieve Inversion of Control (IoC) between classes and their dependencies.
encapsulation? Encapsulation is the process of wrapping data and methods into a single unit, typically a class, and restricting access to some of the object's components.
events? Events are a way for a class to notify other classes or objects when something of interest occurs.
exception handling? Exception handling is a mechanism to handle runtime errors using try, catch, and finally blocks.
garbage collection? Garbage collection automatically manages memory by reclaiming unused objects to free up resources.
inheritance? Inheritance is a feature that allows a class to inherit methods and properties from another class.
LINQ? LINQ (Language Integrated Query) is a feature that allows querying data from different sources (e.g., collections, databases) in a consistent manner.
MVC MVC (Model-View-Controller) is a design pattern that separates an application into three interconnected components: Model (data), View (UI), and Controller (logic).
namespace? A namespace is a container for classes and other types, used to organize code and prevent naming conflicts.
overloading vs overriding? Overloading allows multiple methods with the same name but different parameters, while overriding allows a subclass to provide a specific implementation of a method in its parent class.
polymorphism? Polymorphism allows methods to have the same name but behave differently based on the object that calls them.
sealed class? A sealed class is a class that cannot be inherited by other classes.
serialization? Serialization is the process of converting an object into a format that can be stored or transmitted and later reconstructed.
static class? A static class is a class that cannot be instantiated and can only contain static members.
struct vs class? A struct is a value type and is stored on the stack, while a class is a reference type and is stored on the heap.
== vs .Equals()? "==" checks for reference equality for reference types, while ".Equals()" checks for value equality.
using statement? The "using" statement is used to include namespaces in a program or to ensure the proper disposal of resources.
value types vs reference types? Value types store data directly, while reference types store references to the data's memory address.
asynchronous programming? Asynchronous programming allows tasks to run independently of the main program flow, improving responsiveness and performance.
attributes? Attributes provide metadata about the program elements, such as classes, methods, and properties, and can be used to control their behavior.
data annotations? Data annotations are attributes used to configure model properties, such as validation rules, in frameworks like Entity Framework.
dynamic keyword? The dynamic keyword allows variables to bypass compile-time type checking, enabling runtime type resolution.
extension methods? Extension methods allow you to add new methods to existing types without modifying their source code.
generics? Generics allow you to define classes, methods, and data structures with a placeholder for the type of data they store or use.
indexers? Indexers allow objects to be indexed like arrays, providing a way to access data in a class using array-like syntax.
iterators? Iterators are methods that use the yield keyword to return elements one at a time in a collection.
partial classes? Partial classes allow a class to be split into multiple files, making it easier to manage and maintain large classes.
reflection? Reflection provides the ability to inspect and interact with metadata and types at runtime.
threading? Threading allows multiple threads to run concurrently, enabling parallel execution of tasks.
tuples? Tuples are data structures that can hold multiple values of different types in a single object.
unsafe code? Unsafe code allows the use of pointers and direct memory manipulation in C#.
virtual methods? Virtual methods allow derived classes to override the implementation of a method defined in a base class.
yield keyword? The yield keyword is used in iterators to return each element of a collection one at a time.
ESLint/StyleCop/Code Analysis? Use free static analysis tools like StyleCop (for C#) to check for coding standards and best practices. These tools can be run locally or integrated into your CI/CD pipeline.
