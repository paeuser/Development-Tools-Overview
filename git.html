How to Check In and Push This Project with Git

  1. Open a terminal in the project directory:
    cd C:\Play\1Epic2005\Applications\Development-Tools-Overview
  2. Check the status of your files:
    git status
  3. Add all changes to staging:
    git add .
  4. Commit your changes:
    git commit -m "Describe your changes here"
  5. Push to the remote repository:
    git push

Notes

  • Make sure you have set up the remote repository (e.g., on GitHub) and have the correct permissions.
  • If this is your first push, you may need to set the upstream branch:
    git push --set-upstream origin main
  • Replace main with your branch name if different.

Scenario: PR Approvals and Merging Develop

Scenario: You created a pull request (PR). Some people approved it, but you were told not to complete it yet. Later, you see a warning about your branch being out of date. Paul says:

Paul Williams (Wednesday):
It's not necessary to merge Develop into this branch unless it contains changes that will impact this PR. Merging Develop unnecessarily just resets the votes.

What does this mean?

  • Only merge develop into your branch if you need updates from develop (e.g., bug fixes, new features, or to resolve conflicts).
  • If you merge develop into your branch, PR approvals may be reset and reviewers will need to approve again.
  • If you do not merge develop, your PR remains as-is and votes are preserved. Any conflicts will be resolved when the PR is eventually merged.

Best Practice

  • Do not merge develop into your branch unless necessary.
  • Communicate with your team if you are unsure whether you need to update your branch.

Summary

Only update your branch with develop if it is required for your changes to work or to resolve conflicts. Otherwise, leave your branch as-is to avoid resetting PR approvals.
