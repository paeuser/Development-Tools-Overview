Authentication and Authorization: JWT & OAuth

Authentication vs Authorization

  • Authentication: Verifies who a user is (identity).
  • Authorization: Determines what a user can do (permissions/access).

What is JWT (JSON Web Token)?

  • A compact, URL-safe token format for securely transmitting information between parties as a JSON object.
  • Commonly used for authentication in web applications (stateless sessions).
  • Consists of three parts: header, payload, and signature.
  • Example: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

What is OAuth?

  • An open standard for access delegation (authorization), commonly used for token-based access to APIs.
  • Allows users to grant third-party applications access to their resources without sharing passwords.
  • OAuth 2.0 is the most widely used version.
  • Common in social login ("Sign in with Google/Facebook") and API integrations.

How They Work Together

  • OAuth is used to obtain an access token (often a JWT) after user authentication.
  • The access token is sent with API requests to prove identity and permissions.

Example JWT Payload

{
  "sub": "1234567890",
  "name": "John Doe",
  "role": "admin",
  "exp": 1718025600
}

Benefits

  • Stateless authentication (no server session storage needed)
  • Scalable and secure for APIs and SPAs
  • Widely supported and easy to integrate
