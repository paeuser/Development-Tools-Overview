Common Vulnerabilities: XSS, CSRF, SQL Injection

Cross-Site Scripting (XSS)

  • What is it? XSS allows attackers to inject malicious scripts into web pages viewed by other users.
  • Impact: Can steal cookies, session tokens, or perform actions as the user.
  • Prevention: Always escape user input, use Content Security Policy (CSP), and sanitize data before rendering.

Cross-Site Request Forgery (CSRF)

  • What is it? CSRF tricks a user into submitting unwanted actions on a web app where they're authenticated.
  • Impact: Can change user data, make purchases, or perform admin actions without consent.
  • Prevention: Use anti-CSRF tokens, check referer headers, and require re-authentication for sensitive actions.

SQL Injection

  • What is it? SQL Injection lets attackers run arbitrary SQL queries by injecting code into input fields.
  • Impact: Can read, modify, or delete database data; sometimes gain full control of the server.
  • Prevention: Always use parameterized queries or prepared statements, validate and sanitize input, and limit database permissions.

Best Practices

  • Validate and sanitize all user input.
  • Use security libraries and frameworks that protect against these attacks.
  • Keep software and dependencies up to date.
  • Educate developers about secure coding practices.
