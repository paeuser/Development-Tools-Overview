Assurity ECS API – Backend

What is this?
The Assurity ECS API is the backend service for the Assurity platform. It powers all business logic, data management, and integrations, acting as the core engine for the system.

Who uses it?
Developers and system integrators use the API to build, extend, and automate the Assurity platform for their organizations or clients.
Why is it important?
It provides secure, reliable, and scalable endpoints for managing products, users, configurations, and integrations with external services.
Main Backend Features
  • RESTful API endpoints for configuration, product, and user management
  • White label support for custom branding
  • Authentication and authorization (token-based security)
  • Integration with email and e-signature providers
  • Comprehensive error handling and logging
What is White Label?
White labeling lets organizations customize the platform’s branding (logo, colors, styles) so it appears as their own product.
Technologies & Architecture Details (Backend)
  • .NET Core (C#) – Version: 6.0 (or higher, see Assurity.ECS.sln for details)
  • ASP.NET Web API – for RESTful endpoints
  • Entity Framework Core – for database access
  • SQL Server – as the primary database
  • Authentication: JWT (JSON Web Token) Bearer authentication for secure API access
  • Authorization: Role-based access control (RBAC) for different user types
  • Caching: In-memory caching (MemoryCache) for performance optimization on frequently accessed data
  • Logging & Monitoring: New Relic, NLog for diagnostics and monitoring
  • Integration: Email and E-signature services
