Web Services: REST and SOAP

What is a Web Service?

A web service is a way for applications to communicate with each other over the internet using standard protocols. The two most common types are REST and SOAP.

REST (Representational State Transfer)

  • Uses standard HTTP methods (GET, POST, PUT, DELETE).
  • Stateless: each request contains all the information needed.
  • Data is usually sent and received as JSON or XML.
  • Resource-based URLs (e.g., /api/products/1).
  • Simple, scalable, and widely used for modern web APIs.

SOAP (Simple Object Access Protocol)

  • Uses XML for all messages.
  • Relies on a strict messaging structure (envelope, header, body).
  • Supports advanced features like security, transactions, and ACID compliance.
  • Can use protocols other than HTTP (e.g., SMTP).
  • Common in enterprise and legacy systems.

Key Differences

  • REST: Lightweight, easy to use, best for web/mobile APIs.
  • SOAP: Heavyweight, more features, best for complex enterprise needs.

Example REST Request

GET /api/users/123 HTTP/1.1
Host: example.com

Example SOAP Request

<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Body>
    <GetUser>
      <UserId>123</UserId>
    </GetUser>
  </soap:Body>
</soap:Envelope>
