✅ What should you use as an admin managing many clients?

Here’s a simple 3-tool setup that works for 90% of real-world admin needs:

🔹 1. Azure PowerShell (Az module)

  • Best for automating tasks (create users, VMs, policies, etc.)
  • Use it with scripts locally or in PowerShell

✅ Use when:
You need to script or automate things in Azure across multiple clients.

🔹 2. Azure CLI (az command)

  • Simple command-line tool, works in any OS
  • Easier syntax than PowerShell for many

✅ Use when:
You want quick commands or use Linux/macOS.

🔹 3. Azure Lighthouse

  • Gives you access to many clients’ environments from one login
  • You don’t need to keep switching accounts

✅ Use when:
You manage multiple customers or tenants.

Bonus: Use Cloud Shell

  • Already has CLI & PowerShell installed
  • Runs in browser — great when you’re on the go

Azure Lighthouse is a service that allows service providers (like Managed Service Providers, or MSPs) to manage Azure resources for multiple customers from a single control plane. It enables cross-tenant management, making it easier and more efficient for MSPs to deliver managed services.

Instead of needing separate credentials or logging into each customer's Azure tenant individually, Azure Lighthouse provides a delegated management capability. This means a customer can grant an MSP permission to manage specific subscriptions or resource groups within their tenant, and the MSP can then access and manage those resources directly from their own Azure tenant.

Command

What it Does

Get-AzResourceGroup

Lists all resource groups in your current Azure subscription.

Get-AzSubscription

Shows all Azure subscriptions you have access to.

Set-AzContext

Switches to a specific subscription or tenant context.

New-AzResourceGroup

Creates a new resource group in a specified Azure region.

Remove-AzResourceGroup

Deletes a resource group and all resources within it.

Get-AzVM

Lists all virtual machines in your current subscription.

New-AzVM

Creates a new virtual machine in a specified resource group and location.

Connect-AzAccount

Logs you into your Azure account in the PowerShell session.

Passing parameter

Get-AzResourceGroup | Select-Object ResourceGroupName, ResourceId

Get-AzResourceGroup: This cmdlet fetches all the Azure resource groups in your current subscription. Each resource group is a container that holds related Azure resources (like VMs, storage accounts, etc.).

The pipe symbol | takes the output (list of resource groups) from Get-AzResourceGroup and sends it as input to the next command.

Select-Object ResourceGroupName, ResourceId: This filters the properties of each resource group object, showing only the ResourceGroupName (the friendly name you gave the group) and the ResourceId (the unique full Azure identifier string for that group).

In short:

You list all resource groups but only display their name and unique ID, making the output cleaner and easier to read or use later.

🔧 Register-PSRepository

📘 What it does:

The Register-PSRepository cmdlet adds a PowerShell repository to your session, allowing you to install modules from it using commands like Install-Module.

Think of it as telling PowerShell where to find packages (like NuGet for .NET or pip for Python).

ModuleBuilder is a PowerShell project template and build tool that helps you create and structure reusable PowerShell modules. It lets you organize your code cleanly using folders like Public, Private, and Tests, and then automatically builds a .psm1 file by combining your functions. It supports versioning, testing, and publishing, making it easier to maintain and share your module. This tool is ideal if you're developing modules that will grow over time or be shared with others (like on PowerShell Gallery). It works well with source control systems like Git and automation tools like Azure DevOps.
