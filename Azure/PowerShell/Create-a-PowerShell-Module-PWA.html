1. Create a Script File with Functions

Start by writing one or more functions in a .psm1 file:

# MyTools.psm1
function Say-Hello {
    param($Name)
    "Hello, $Name!"
}

function Add-Numbers {
    param($a, $b)
    return $a + $b
}

You now have the code for your module.

2. Create a Folder and Save the Module

Your module must go into a folder with the same name as the .psm1 file:

MyTools\

üì§ How to Publish a Module from Local Machine to Gallery

  1. Create a Module Manifest: Use New-ModuleManifest to create a .psd1 file with metadata.
  2. Ensure Your Module Folder Structure: Your folder (e.g., MyTools) contains MyTools.psm1 and the manifest MyTools.psd1.
  3. Get a NuGet API Key: Sign into PowerShell Gallery, generate an API key, and keep it safe.
  4. Install or Update PowerShellGet: Run Install-Module -Name PowerShellGet -Force in an admin PowerShell.
  5. Publish the Module: Run Publish-Module -Path "C:\Path\To\MyTools" -NuGetApiKey "YOUR_API_KEY" replacing path and API key.
  6. List All Commands in Your Module: Run Get-Command -Module MyTools to see all available functions and cmdlets in your module.
  7. Verify Installation: On any machine, run Install-Module -Name MyTools -Scope CurrentUser to install your module.
Following these steps will share your module with the PowerShell community via the Gallery!
