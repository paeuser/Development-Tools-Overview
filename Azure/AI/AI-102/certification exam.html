AI-102 Exam Preparation Notes

Certification

Microsoft Certified: Azure AI Engineer Associate
Design and implement an Azure AI solution using Azure AI services, Azure AI Search, and Azure OpenAI.

Azure AI Services Overview

Azure AI Services are building blocks of AI functionality you can integrate into your applications. Learn how to provision, secure, monitor, and deploy Azure AI Services resources and use them to build intelligent solutions.

Common Azure AI Services

ServiceDescription
Azure OpenAIProvides access to OpenAI generative AI models (GPT, DALL-E) within a scalable and secure Azure cloud service.
Azure AI VisionAPIs and models for computer vision: detect objects, generate captions, read text in images.
Azure AI SpeechAPIs for text-to-speech, speech-to-text, speaker recognition, and translation.
Azure AI LanguageAnalyze text, extract entities, sentiment, summarization, build conversational models, Q&A.
Azure AI Foundry Content SafetyAdvanced algorithms to flag potentially offensive, risky, or undesirable content in images and text.
Azure AI TranslatorTranslate text between many languages using state-of-the-art models.
Azure AI FaceDetect, analyze, and recognize human faces. Some features are restricted for privacy/safety.
Azure AI Custom VisionTrain and use custom computer vision models for image classification and object detection.
Azure AI Document IntelligenceExtract fields from complex documents (invoices, receipts, forms) using pre-built or custom models.
Azure AI Content UnderstandingMulti-modal content analysis for extracting data from forms, documents, images, video, and audio.
Azure AI SearchPipeline of AI skills to extract information and create a searchable index, often used for vector search and grounding prompts for generative AI.

Provisioning and Using Azure AI Services

  • Create Azure AI resources in your subscription and use client code to consume them.
  • Some services have web-based visual interfaces for configuration and testing (e.g., Custom Vision).
  • You can provision resources via the Azure portal, BICEP/ARM templates, or Azure CLI.
  • For medium/large projects, use an Azure Foundry hub to centralize access, cost management, and resource sharing.
Note: Provisioning Azure AI services as part of a Foundry hub is recommended for larger projects to simplify management and cost control.

Single Service vs. Multi-Service Resource

  • Most Azure AI services (Vision, Language, etc.) can be provisioned as standalone resources (often with a free tier).
  • Standalone resources provide their own endpoint and keys for secure access.
  • Alternatively, you can provision a multi-service resource that includes multiple AI services in one resource:
    • Azure OpenAI
    • Azure AI Speech
    • Azure AI Vision
    • Azure AI Language
    • Azure AI Foundry Content Safety
    • Azure AI Translator
    • Azure AI Document Intelligence
    • Azure AI Content Understanding
  • Multi-service resources make it easier to manage apps using multiple AI capabilities.
Tip: When provisioning, select the Azure AI services resource type with the latest icon. Avoid the older "Azure Cognitive Services" resource type for new projects.

Regional Availability

  • Some services/models are only available in certain Azure regions.
  • Check the product availability table and model availability table for details.

Cost

  • Azure AI services are charged based on usage, with different pricing for each service.
  • Use the Azure pricing calculator to estimate costs for your expected usage.
  • Review the Azure AI services pricing documentation for up-to-date pricing info.
