Azure DevOps & GitHub CI/CD Pipelines - 50 Interview Questions & Answers

  1. What is CI/CD?
    CI/CD stands for Continuous Integration and Continuous Deployment/Delivery, a set of practices for automating the building, testing, and deployment of applications.
  2. What is Azure DevOps?
    Azure DevOps is a suite of development tools by Microsoft for source control, build/release pipelines, project management, and more.
  3. What is a pipeline in Azure DevOps?
    A pipeline automates the process of building, testing, and deploying code.
  4. What is GitHub Actions?
    GitHub Actions is a CI/CD and automation platform built into GitHub, allowing you to automate workflows directly from your repository.
  5. What is a YAML pipeline?
    A pipeline defined as code using YAML syntax, enabling versioning and easy reuse.
  6. What is the difference between build and release pipelines?
    Build pipelines compile and test code; release pipelines deploy built artifacts to environments.
  7. What are pipeline agents?
    Agents are machines that run pipeline jobs (hosted by Microsoft or self-hosted).
  8. What is an artifact?
    An artifact is a file or set of files produced by a build, used as input for deployments.
  9. How do you trigger a pipeline automatically?
    By configuring triggers such as push, pull request, or schedule in the pipeline definition.
  10. What is a stage in a pipeline?
    A logical boundary in a pipeline representing a phase (e.g., build, test, deploy).
  11. What is a job in a pipeline?
    A job is a collection of steps run sequentially on the same agent.
  12. What is a step in a pipeline?
    A step is a single task, script, or action within a job.
  13. How do you secure secrets in pipelines?
    By using secret variables, Azure Key Vault, or GitHub Secrets to store sensitive data.
  14. What is environment in Azure DevOps?
    An environment represents a target for deployment, such as dev, test, or production.
  15. What is a service connection?
    A secure connection to external services/resources (e.g., Azure, Docker Hub) used in pipelines.
  16. How do you implement approvals in pipelines?
    By configuring manual approval gates before or after stages/environments.
  17. What is a self-hosted agent?
    An agent you install and manage on your own infrastructure for custom builds or deployments.
  18. What is a hosted agent?
    An agent provided and managed by Microsoft or GitHub, running in the cloud.
  19. How do you reuse pipeline code?
    By using templates (YAML includes) or composite actions in GitHub Actions.
  20. What is a pipeline variable?
    A value used in pipelines for configuration, which can be set at queue time, in YAML, or in the UI.
  21. How do you handle pipeline failures?
    By reviewing logs, using retry strategies, and setting up notifications for failed runs.
  22. What is a pull request validation pipeline?
    A pipeline that runs automatically to validate code changes in a pull request before merging.
  23. How do you deploy to Azure from a pipeline?
    By using Azure CLI, ARM/Bicep templates, or built-in Azure deployment tasks/actions.
  24. What is a matrix build?
    Running the same job with different parameters (e.g., OS, language version) in parallel.
  25. What is a pipeline artifact?
    A file or set of files published by a pipeline for sharing between jobs or stages.
  26. What is a GitHub workflow?
    A YAML file in the .github/workflows directory that defines automation for your repository.
  27. How do you cache dependencies in pipelines?
    By using cache tasks/actions to speed up builds by reusing previously downloaded dependencies.
  28. What is a deployment strategy?
    A method for rolling out changes, such as blue-green, canary, or rolling deployments.
  29. What is infrastructure as code (IaC)?
    Managing infrastructure using code (e.g., ARM, Bicep, Terraform) for repeatable deployments.
  30. How do you integrate tests in CI/CD?
    By adding test steps/tasks in the pipeline to run unit, integration, or end-to-end tests.
  31. What is a GitHub Action?
    A reusable unit of automation in GitHub Actions, defined in a repository or published on the Marketplace.
  32. How do you rollback a deployment?
    By redeploying a previous artifact or using deployment slots/environments.
  33. What is a pipeline template?
    A reusable YAML file that defines common pipeline logic for inclusion in multiple pipelines.
  34. What is a build trigger?
    An event that starts a build, such as a code push, PR, or schedule.
  35. How do you monitor pipeline runs?
    By using dashboards, logs, and notifications in Azure DevOps or GitHub.
  36. What is a deployment slot?
    A live environment for staging deployments before swapping to production (e.g., in Azure App Service).
  37. How do you use Key Vault in pipelines?
    By linking Azure Key Vault to your pipeline and referencing secrets as variables.
  38. What is a conditional step?
    A step that runs only if certain conditions are met (e.g., based on variables or previous step results).
  39. What is a GitHub runner?
    A machine that executes GitHub Actions workflows (hosted or self-hosted).
  40. How do you deploy to multiple environments?
    By defining multiple stages/environments in your pipeline and using approvals or gates.
  41. What is a pipeline badge?
    A status indicator (image) showing the latest pipeline run result, embeddable in README files.
  42. How do you handle secrets in GitHub Actions?
    By storing them in GitHub Secrets and referencing them in workflow YAML.
  43. What is a release gate?
    A precondition (e.g., approval, query, or check) that must be met before a release proceeds.
  44. How do you use environment variables in pipelines?
    By defining them in YAML, pipeline settings, or as secrets, and referencing them in steps.
  45. What is a build artifact?
    The output of a build process, such as binaries, packages, or container images.
  46. How do you integrate code quality checks?
    By adding static analysis, linting, or security scanning tasks to your pipeline.
  47. What is a pipeline schedule?
    A configuration to run pipelines at specific times (e.g., nightly builds).
  48. How do you use templates in GitHub Actions?
    By creating composite actions or reusable workflows and referencing them in workflow YAML.
  49. What is a deployment job?
    A job that deploys artifacts to an environment, often with rollback and approval support.
  50. How do you integrate notifications?
    By configuring email, Teams, Slack, or other integrations for pipeline events.
  51. What is a canary deployment?
    A deployment strategy where changes are rolled out to a small subset of users before full rollout.
