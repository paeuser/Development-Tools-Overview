Azure Vocabulary: Tenant & Key Vault

Term: tenant

Definition: In Azure, a tenant is a dedicated, isolated instance of Azure Active Directory (Azure AD) that an organization receives when it signs up for a Microsoft cloud service such as Azure, Microsoft 365, or Intune.

  • The tenant represents your organization and is used to manage users, groups, and access to resources across Microsoft cloud services.
  • Each tenant is unique and separate from other tenants, providing security and isolation for your organization's data and identity management.
  • Tenants are identified by a unique domain name (e.g., yourcompany.onmicrosoft.com) and a globally unique identifier (GUID).

Example: If your company signs up for Azure, you get a tenant (e.g., contoso.onmicrosoft.com) that you use to manage all your users and resources in Azure.

Term: Azure Key Vault

Definition: Azure Key Vault is a cloud service for securely storing and managing sensitive information such as secrets, encryption keys, and certificates.

  • Helps safeguard cryptographic keys and secrets used by cloud applications and services.
  • Supports secure storage and tight access control for tokens, passwords, API keys, and other secrets.
  • Enables you to control access using Azure Active Directory and policies.
  • Provides logging and monitoring for access and usage.
  • Integrates with other Azure services for secure key management and secret retrieval.

Example: You can store a database connection string or an SSL certificate in Azure Key Vault and allow only your application to access it securely at runtime.
