Understanding Azure User Accounts

Your Personal Microsoft Account (MSA)

eric-cabrera@live.com is your personal Microsoft Account (MSA). This is the email and identity you used to sign up for Azure.

Default Directory

When you create an Azure subscription with a personal Microsoft Account, Azure automatically creates a directory (tenant) for you, called a “default directory”.

This directory contains a user that corresponds to your MSA (eric-cabrera@live.com), but sometimes the portal will show this account as a guest user in another tenant, especially if you are invited to collaborate on other Azure tenants.

Guest User Accounts

The account "eric cabrera (Guest)" with the email eric-cabrera_live.com#EXT#@ericcabreralive.onmicrosoft.com represents a guest user in the Azure Active Directory tenant named ericcabreralive.onmicrosoft.com.

  • This guest account is created when your personal Microsoft Account (eric-cabrera@live.com) is invited to collaborate in another organization’s Azure tenant.
  • Azure AD modifies the email to include #EXT# and replaces the @ symbol to indicate it is an external user.
  • Because this is a guest account, it has limited permissions compared to your main user account and is distinct from your primary login, which owns your Azure subscription and has full administrative rights.
