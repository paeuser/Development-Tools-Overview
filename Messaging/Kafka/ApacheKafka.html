What is Apache Kafka?

Video Reference

For a long time, information was stored in databases—think of the world in terms of users and things. This worked well for decades, but today it's often better to think in terms of events. An event is a record of something that happened at a specific time.

Storing events in traditional databases can be cumbersome. Instead, Kafka uses a log: a sequence of events with timestamps and descriptions. Logs are easy to scale and reason about.

Apache Kafka manages these logs using topics—durable, ordered collections of events. Topics can store data for hours, days, or indefinitely, and are replicated for reliability.

Each event represents something that happened in the business, like a user changing their shipping address or a train unloading cargo. Kafka helps you focus on events first, things second.

As applications grow, they often become too complex for a single developer to understand. The modern trend is to break them into many small services, each focused and easy to manage. These services communicate by producing and consuming events through Kafka topics.

This event-driven approach enables new systems, like real-time analytics dashboards, to consume and analyze data as it arrives.

If you understand these four ideas—events, logs, topics, and small services—you'll have a good sense of what Kafka is and how to build architectures around it.

Kafka also integrates with other systems. For example, if you have an existing database or a SaaS product, Kafka Connect can bring that data into Kafka topics, making it available for new services to consume.

For real-time analysis without writing custom code, Confluent offers KSQL, a SQL-like language for querying Kafka topics. Confluent Platform is a distribution of Kafka with extra tools and managed services, including Confluent Cloud for serverless Kafka in the cloud.

Kafka in a Nutshell

  • Publish (produce): Send streams of events from apps, devices, or systems.
  • Store: Keep events durably and reliably in topics.
  • Subscribe (consume): Read events in real time for processing, analytics, or integration.

Core Kafka Components

  • Producers: Send events to Kafka topics.
  • Topics: Durable, ordered logs of events—can be small or massive, short- or long-lived.
  • Consumers: Read events from topics for processing or analytics.
  • Kafka Connect: Integrates Kafka with external systems (databases, SaaS, search, etc.) using ready-made connectors.
  • Kafka Streams: Java API for building real-time stream processing apps (grouping, aggregating, filtering, joining, etc.).
  • KSQL: SQL-like language for real-time analytics on Kafka topics—enables queries and transformations without code.
  • Confluent Platform: Distribution of Kafka with extra tools, managed services (Confluent Cloud), and enterprise features.

How Kafka Fits in Modern Architectures

  • Enables event-driven architectures—many small services communicate by producing and consuming events.
  • Supports real-time analytics and dashboards by making data available as soon as events happen.
  • Kafka Connect makes it easy to integrate legacy systems and external data sources.
  • Kafka Streams and KSQL allow powerful, scalable processing and transformation of event data.
  • Confluent Platform and Confluent Cloud simplify running Kafka at scale, with managed infrastructure and extra tools.
