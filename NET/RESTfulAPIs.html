What is a RESTful API? A RESTful API is an application programming interface that adheres to the principles of REST (Representational State Transfer), enabling interaction with web services using standard HTTP methods.
What are the HTTP methods used in RESTful APIs? The common HTTP methods are GET, POST, PUT, DELETE, PATCH, and OPTIONS, each serving a specific purpose in CRUD operations.
What is statelessness in REST? Statelessness means that each request from a client to a server must contain all the information needed to understand and process the request, with no reliance on stored context on the server.
What is a resource in REST? A resource is any piece of information that can be named and manipulated via a RESTful API, typically represented by a URI (Uniform Resource Identifier).
What is JSON in RESTful APIs? JSON (JavaScript Object Notation) is a lightweight data-interchange format commonly used for transmitting data between a client and a server in RESTful APIs.
What is HATEOAS? HATEOAS (Hypermedia as the Engine of Application State) is a constraint of REST that allows clients to dynamically navigate resources through hyperlinks provided in responses.
What is versioning in RESTful APIs? Versioning is the practice of managing changes to an API by providing different versions, often indicated in the URI or headers, to ensure backward compatibility.
What is an API gateway? An API gateway is a server that acts as an entry point for API requests, handling tasks such as request routing, authentication, rate limiting, and monitoring.
What is CORS in RESTful APIs? CORS (Cross-Origin Resource Sharing) is a mechanism that allows restricted resources on a web page to be requested from another domain outside the domain from which the resource originated.
What is an API key? An API key is a unique identifier used to authenticate and authorize access to an API, ensuring secure communication between the client and the server.
Define Messaging in terms of RESTful web services. Messaging refers to the exchange of information: the client sends an HTTP request, and the server responds with an HTTP Response, both containing data and metadata.
Can you tell the disadvantages of RESTful web services? Disadvantages include challenges in maintaining sessions due to statelessness (client responsibility) and the need for explicit implementation of security measures as REST itself doesn't impose them.
What are HTTP Status codes? HTTP Status codes are standard numerical codes indicating the outcome of an HTTP request (e.g., 200 OK for success, 404 Not Found for client error, 500 Internal Server Error for server error).
What do you understand by JAX-RS? JAX-RS (Java API for RESTful Web Services) is a Java-based specification defined by Java EE for implementing RESTful services, simplifying development through annotations.
What are the features of RESTful Web Services? Features include client-server model, HTTP protocol usage, messaging, URI-identified resources, statelessness, cacheability, layered system, and a uniform interface.
What is URI? URI (Uniform Resource Identifier) is a string of characters used to identify a resource on the internet. It can be a URN (name-based) or a URL (location-based).
Differentiate between SOAP and REST. SOAP is a protocol (strict, XML-only, less flexible) while REST is an architectural style (flexible, supports various formats like JSON, uses standard HTTP, generally faster).
While creating URI for web services, what are the best practices that needs to be followed? Best practices include using plural nouns for collections, hyphens for readability, lowercase, maintaining backward compatibility, using appropriate HTTP methods (not in URI), and forward slashes for hierarchy.
What are the best practices to develop RESTful web services? Key practices involve preferring JSON for data exchange, setting appropriate Content-Type headers, proper use of HTTP methods, implementing robust error handling with status codes, and security.
ASP.NET and APIs Specific Questions
What is ASP.NET Web API? ASP.NET Web API is a framework for building HTTP services that can be consumed by a broad range of clients including browsers, mobile devices, and other applications. It is an ideal platform for building RESTful applications on the .NET Framework (or .NET Core/.NET).
How does ASP.NET Web API differ from ASP.NET MVC? ASP.NET Web API is designed specifically for building HTTP services that return data, typically in JSON or XML, making it ideal for APIs. ASP.NET MVC is primarily for building web applications that return views (HTML) to the browser, though it can also return data. In ASP.NET Core, the two frameworks are unified into a single framework.
Explain routing in ASP.NET Web API. Routing in ASP.NET Web API maps incoming HTTP requests to specific controller actions. This can be achieved through convention-based routing (defining patterns in a routing table) or attribute routing (decorating controller actions directly with route attributes like `[Route("api/products")]`).
How do you handle authentication and authorization in ASP.NET Web API? Authentication (verifying user identity) and authorization (determining user permissions) in ASP.NET Web API can be handled using various mechanisms:
  • **Authentication:** Bearer tokens (JWT), OAuth 2.0, API keys, Cookies.
  • **Authorization:** `[Authorize]` attributes (role-based, policy-based), custom authorization filters.
What is Dependency Injection in ASP.NET Core Web API? Dependency Injection (DI) is a software design pattern where components declare their dependencies rather than creating them directly. ASP.NET Core has a built-in DI container that manages the lifetime and creation of services, making APIs more modular, testable, and maintainable.
How can you consume a RESTful API in an ASP.NET application? A RESTful API can be consumed in an ASP.NET application primarily using the `HttpClient` class. This class provides methods for sending HTTP requests (GET, POST, PUT, DELETE) and handling responses asynchronously. Libraries like Newtonsoft.Json are often used for JSON serialization/deserialization.
What is Swagger/OpenAPI, and how is it used with ASP.NET Web API? Swagger (now part of OpenAPI Specification) is a framework of API development tools for describing, producing, consuming, and visualizing RESTful web services. With ASP.NET Web API (especially ASP.NET Core), tools like Swashbuckle.AspNetCore are used to automatically generate OpenAPI documentation from your API code, which can then be used to create interactive API reference pages (Swagger UI).
Explain the use of `HttpClient` in .NET for consuming APIs. `HttpClient` is a class in .NET used to send HTTP requests and receive HTTP responses from a resource identified by a URI. It provides methods like `GetAsync()`, `PostAsJsonAsync()`, `PutAsJsonAsync()`, and `DeleteAsync()` for interacting with RESTful APIs, often used within a `using` statement or as a singleton instance to manage connections efficiently.
