What is .NET Core? .NET Core is a cross-platform, open-source framework for building modern, cloud-based, and internet-connected applications.
What is Entity Framework Core? Entity Framework Core (EF Core) is a lightweight, extensible, and cross-platform version of the popular Entity Framework data access technology.
What are migrations in EF Core? Migrations in EF Core are a way to incrementally update the database schema to keep it in sync with the application's data model.
What is DbContext? DbContext is the primary class in EF Core that interacts with the database and manages entity objects during runtime.
What is LINQ in EF Core? LINQ (Language Integrated Query) in EF Core allows querying the database using strongly-typed C# syntax.
What is lazy loading? Lazy loading is a feature in EF Core that delays the loading of related data until it is accessed for the first time.
What is eager loading? Eager loading is a feature in EF Core that loads related data along with the main data in a single query.
What is the difference between Add and Attach in EF Core? Add marks an entity as new and includes it in the database on SaveChanges, while Attach marks an entity as unchanged and does not include it in the database.
What is the purpose of AsNoTracking in EF Core? AsNoTracking is used to query data without tracking it in the DbContext, improving performance for read-only operations.
What is the difference between First and FirstOrDefault in EF Core? First throws an exception if no records are found, while FirstOrDefault returns null if no records are found.
