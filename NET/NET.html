.NET Interview Questions & Answers Comprehensive Guide for Your Interview

.NET Framework Interview Questions

1. How does the .NET framework work?

The .NET Framework is a software development platform for building and running applications on Windows. It consists of the Common Language Runtime (CLR), which handles execution, memory management, and security, and the Framework Class Library (FCL), a vast collection of reusable classes, interfaces, and value types.

2. Explain about major components of the .NET framework.

Major components include:

  • Common Language Runtime (CLR): The execution engine that provides services like garbage collection, security, and exception handling.
  • Framework Class Library (FCL): A comprehensive, object-oriented collection of reusable types that provides functionality for various tasks.
  • Common Type System (CTS): Defines how types are declared, used, and managed in the CLR.
  • Common Language Specification (CLS): A set of rules that define the minimum features required for .NET languages to interoperate.
  • Metadata and Assemblies: Assemblies are the fundamental units of deployment, versioning, reuse, and security in .NET. Metadata describes the types defined within an assembly.

3. What is an EXE and a DLL?

  • EXE (Executable): A file that contains an application's executable code. It has an entry point (e.g., `Main` method) and can be run directly.
  • DLL (Dynamic Link Library): A library file that contains code and data that can be used by multiple programs simultaneously. It does not have an entry point and cannot be run directly. DLLs promote code reuse.

4. What is CTS?

CTS (Common Type System) is a standard that specifies how types are declared, used, and managed in the Common Language Runtime (CLR). It ensures that types defined in different .NET languages can interoperate with each other by providing a common set of data types and operations.

5. Explain CLS

CLS (Common Language Specification) is a subset of the Common Type System (CTS) that defines a set of rules and guidelines that language compilers must follow to produce code that can interoperate with other .NET languages. It ensures cross-language compatibility.

6. What is JIT?

JIT (Just-In-Time) compiler is a component of the Common Language Runtime (CLR) that converts Microsoft Intermediate Language (MSIL) code into native machine code at runtime, just before the code is executed. This allows for platform independence and optimized execution for the specific CPU architecture.

7. What is the difference between int and Int32?

  • int: This is a C# keyword (an alias) for the `System.Int32` type. Its actual size (32-bit or 64-bit) depends on the underlying system and the .NET version, but in modern .NET, it typically maps to `Int32`.
  • Int32: This is the actual .NET Framework type (`System.Int32`) that represents a 32-bit signed integer. It has a fixed size across all .NET languages.

In most practical scenarios, they are interchangeable in C#, but `Int32` is the explicit type from the FCL, while `int` is a language-specific shortcut.

8. Explain the differences between value type and reference type.

  • Value Type: Stores its data directly in its own memory allocation (usually on the stack). When a value type is assigned to another variable, its value is copied. Examples: `int`, `char`, `struct`, `enum`.
  • Reference Type: Stores a reference (memory address) to its data, which is stored on the heap. When a reference type is assigned to another variable, only the reference is copied, pointing to the same data in memory. Examples: `class`, `string`, `array`, `delegate`.

9. What is the difference between managed and unmanaged code?

  • Managed Code: Code that is executed and managed by the Common Language Runtime (CLR). The CLR provides services like automatic garbage collection, type safety, and security. Most C#, VB.NET code is managed.
  • Unmanaged Code: Code that is executed directly by the operating system, outside the control of the CLR. It requires manual memory management and can be less secure. Examples include C++ code, Win32 APIs, and COM components.

10. Explain Microsoft Intermediate Language

Microsoft Intermediate Language (MSIL), also known as Common Intermediate Language (CIL), is a CPU-independent instruction set that .NET language compilers (like C#, VB.NET) translate source code into. This intermediate code is then compiled into native machine code by the JIT compiler at runtime, enabling platform independence.

11. What is an assembly?

An assembly is the fundamental unit of deployment, versioning, reuse, activation scope, and security in .NET. It's a compiled code library that can be either an executable file (.exe) or a dynamic link library (.dll). An assembly contains MSIL code, metadata, and optional resources.

12. Is ASP.NET different from ASP? If yes, explain how?

  • ASP (Active Server Pages): An older, interpreted, server-side scripting environment (using VBScript or JScript) for building dynamic web pages. It's procedural and less structured.
  • ASP.NET: A compiled, object-oriented framework for building dynamic web applications and web services. It's part of the .NET Framework, offering better performance, security, and a rich set of controls and features (like Web Forms, MVC, Web API).

ASP.NET is a complete rewrite and significant advancement over classic ASP.

13. Explain role-based security in .NET.

Role-based security in .NET allows you to control access to resources (methods, files, etc.) based on the roles (groups) that a user belongs to, rather than individual user identities. For example, you can specify that only users in the "Administrators" role can access certain functions.

14. Explain the different types of assembly.

  • Private Assemblies: Assemblies that are deployed with a specific application and are only accessible by that application. They are typically stored in the application's base directory.
  • Shared Assemblies (Strong-Named Assemblies): Assemblies that can be shared by multiple applications. They are installed in the Global Assembly Cache (GAC) and have a strong name (name, version, culture, public key token) to ensure uniqueness and prevent conflicts.

15. What is the order of the events in a page life cycle?

The typical order of events in an ASP.NET page life cycle is:

  1. Page_PreInit: Raised before the `Init` event.
  2. Page_Init: Initializes controls.
  3. Page_InitComplete: Raised after all controls are initialized.
  4. Page_PreLoad: Raised before `Load` event, after postback data is loaded.
  5. Page_Load: Main event for page processing.
  6. Control Events: Events for individual controls (e.g., `Button_Click`).
  7. Page_LoadComplete: Raised after all control events are handled.
  8. Page_PreRender: Raised before the page content is rendered.
  9. Page_PreRenderComplete: Raised after all controls are pre-rendered.
  10. Page_SaveStateComplete: State is saved.
  11. Page_Render: Generates the HTML output.
  12. Page_Unload: Final cleanup.

16. What is a garbage collector?

The Garbage Collector (GC) is a component of the Common Language Runtime (CLR) that automatically manages memory allocation and deallocation for managed objects. It frees up memory occupied by objects that are no longer referenced by the application, preventing memory leaks and simplifying memory management for developers.

17. What is caching?

Caching is a technique used to store frequently accessed data or computed results in a temporary storage location (cache) so that future requests for that data can be served more quickly, without needing to re-fetch or re-compute it. This improves application performance and reduces load on backend systems.

18. Can we apply themes to ASP.NET applications?

Yes, you can apply themes to ASP.NET applications. Themes are collections of properties, style sheets, and images that define the appearance of a web page or control. They provide a consistent look and feel across an application and can be easily switched.

19. Explain MVC.

MVC (Model-View-Controller) is an architectural pattern that separates an application into three main components:

  • Model: Represents the application's data, business logic, and rules.
  • View: Displays the data from the model to the user.
  • Controller: Handles user input, interacts with the model, and selects the appropriate view to display.

This separation of concerns improves maintainability, testability, and scalability.

20. What is cross-page posting?

Cross-page posting in ASP.NET allows a form on one web page to be submitted to a different web page. This is useful when you want to process form data on a separate page, for example, a confirmation page or a payment processing page.

21. What is a delegate in .NET?

A delegate in .NET is a type-safe function pointer. It holds a reference to a method (or methods) and can be used to invoke that method indirectly. Delegates are fundamental to event handling and callback mechanisms in .NET.

22. What are security controls available on ASP.NET?

ASP.NET provides several security controls and features, including:

  • Authentication: Forms Authentication, Windows Authentication.
  • Authorization: URL Authorization, File Authorization, Role-Based Authorization.
  • Membership and Role Management: Built-in providers for managing users and roles.
  • Login Controls: Pre-built controls like `Login`, `LoginView`, `LoginStatus`, `CreateUserWizard`, `PasswordRecovery`.
  • Code Access Security (CAS): (Less common in modern .NET, more in Framework) Restricts what code can do based on its origin.

23. What is boxing and unboxing in .NET?

  • Boxing: The process of converting a value type (e.g., `int`) to a reference type (`object` or any interface type implemented by the value type). This involves wrapping the value type in an `object` on the heap.
  • Unboxing: The explicit process of converting an `object` type back to a value type. This involves extracting the value from the object on the heap.

Both operations can incur performance overhead.

24. What is MIME in .NET?

MIME (Multipurpose Internet Mail Extensions) is a standard that describes the type of content being transmitted over the internet (e.g., `text/html`, `image/jpeg`, `application/json`). In .NET (especially in web applications), MIME types are used to inform browsers or other clients how to interpret the data sent from the server.

25. What is the use of manifest in the .NET framework?

The manifest in a .NET assembly is a block of metadata that describes the assembly itself. It contains information such as the assembly's identity (name, version, culture, public key), a list of all files in the assembly, references to other assemblies, and security permissions required by the assembly.

26. Explain different types of cookies available in ASP.NET?

ASP.NET primarily uses two types of cookies:

  • Session Cookies: These are temporary cookies that exist only for the duration of a user's session. They are stored in the browser's memory and are deleted when the browser is closed.
  • Persistent Cookies: These cookies have an expiration date and are stored on the user's hard drive. They persist across browser sessions and are used for "remember me" functionality, tracking, etc.

27. What is the meaning of CAS in .NET?

CAS (Code Access Security) in .NET Framework is a security mechanism that restricts the ability of managed code to access protected resources and operations based on the code's identity (e.g., where it originated from, who published it). While largely deprecated in modern .NET Core/5+, it was a key security feature in the full .NET Framework.

28. What is the appSettings section in the web.config file?

The appSettings section in the `web.config` file is used to store custom application-specific configuration settings as key-value pairs. This allows developers to easily change application behavior without recompiling the code (e.g., database connection strings, API keys, feature flags).

29. What is the difference between an abstract class and an interface?

  • Abstract Class: Can have abstract (unimplemented) and concrete (implemented) methods. Can have fields, properties, constructors. Supports single inheritance. Used for "is-a" relationships where some implementation is shared.
  • Interface: Can only declare methods, properties, events, and indexers (no implementation until C# 8 default interface methods). Cannot have fields or constructors. Supports multiple inheritance. Used for "can-do" relationships, defining a contract.

30. What are the types of memories supported in the .NET framework?

The .NET Framework primarily uses two types of memory:

  • Stack: Used for storing value types, method parameters, and local variables. Memory is allocated and deallocated in a LIFO (Last-In, First-Out) manner.
  • Heap: Used for storing reference types (objects). Memory is allocated dynamically and managed by the Garbage Collector.

31. Explain localization and globalization.

  • Globalization: The process of designing and developing applications that support multiple cultures and regions. This involves designing the application to handle different date/time formats, number formats, currency, sorting rules, etc.
  • Localization: The process of adapting a globalized application to a specific culture or region. This involves translating UI text, images, and other resources to the target language and culture.

32. What are the parameters that control the connection pooling behaviors?

Connection pooling in ADO.NET is controlled by parameters in the connection string, such as:

  • Pooling=true/false: Enables/disables pooling.
  • Min Pool Size: Minimum number of connections to keep in the pool.
  • Max Pool Size: Maximum number of connections allowed in the pool.
  • Connection Lifetime: Maximum time (in seconds) a connection can live in the pool.
  • Load Balance Timeout: Time (in seconds) after which a connection is removed from the pool if it has been idle.

33. What are MDI and SDI?

  • MDI (Multiple Document Interface): An application design where a single parent window contains multiple child windows. Each child window can display a different document or view (e.g., older versions of Microsoft Word with multiple document windows inside one main window).
  • SDI (Single Document Interface): An application design where each document or view is opened in a separate, independent top-level window (e.g., modern web browsers, Notepad).

34. Explain the different parts of an Assembly.

An assembly in .NET typically consists of four main parts:

  • Manifest: Contains metadata about the assembly itself (identity, files, referenced assemblies, security permissions).
  • Metadata: Describes the types (classes, interfaces, structs, enums) and members (methods, properties, fields) defined within the assembly.
  • MSIL (Microsoft Intermediate Language): The platform-independent executable code.
  • Resources: Non-executable data like images, strings, XML files, etc., embedded within the assembly.

.NET Core Interview Questions

1. What are Universal Windows Platform (UWP) Apps in .Net Core?

UWP (Universal Windows Platform) Apps are applications that run on all Windows-powered devices, including PCs, Xbox, HoloLens, and Surface Hub, using a single codebase. While UWP apps can leverage .NET Core components, they are primarily built using the Windows SDK and XAML, and are distributed via the Microsoft Store.

2. Write a program to calculate the addition of two numbers.

Here's a simple C# program:


using System;

public class Addition
{
    public static void Main(string[] args)
    {
        int num1 = 10;
        int num2 = 20;
        int sum = num1 + num2;
        Console.WriteLine($"The sum of {num1} and {num2} is: {sum}");
    }
}

3. Explain about .NET Core Components.

.NET Core (now simply .NET) components include:

  • .NET Runtime (CoreCLR): The execution engine, providing services like garbage collection, JIT compilation, and type system.
  • ASP.NET Core: A cross-platform framework for building modern web apps and APIs.
  • Entity Framework Core: An ORM for data access.
  • .NET SDK: A set of tools and libraries for developing, building, and running .NET applications.
  • Base Class Library (BCL): A subset of the FCL, providing fundamental types and functionality.

4. What is middleware in .NET core?

Middleware in ASP.NET Core is software that is assembled into an application pipeline to handle requests and responses. Each component chooses whether to pass the request to the next component in the pipeline, or short-circuit the pipeline. Examples include authentication, routing, and error handling middleware.

5. Differentiate .NET Core vs .NET framework.

  • .NET Framework: Windows-only, older, larger, full FCL, primarily for desktop and web (ASP.NET Web Forms/MVC 5).
  • .NET Core (now .NET): Cross-platform (Windows, Linux, macOS), open-source, modular, smaller footprint, command-line tooling, optimized for modern cloud-native apps, microservices, and containers.

6. Explain Explicit Compilation (Ahead Of Time compilation).

Ahead-Of-Time (AOT) compilation is a process where code is compiled into native machine code before runtime, typically during the build process. This contrasts with JIT compilation, which happens at runtime. AOT compilation can result in faster startup times and smaller deployment sizes, as the compilation overhead is removed from the runtime. .NET Native (for UWP) and Blazor WebAssembly use AOT.

7. What is MEF?

MEF (Managed Extensibility Framework) is a library for creating lightweight, extensible applications. It allows developers to discover and use extensions without configuration. It's often used for plugin architectures and dependency injection, though its use has somewhat declined with the rise of built-in DI containers in .NET Core.

8. In what situations .NET Core and .NET Standard Class Library project types will be used?

  • .NET Core Project: Used for building executable applications (console, web, desktop) that target a specific .NET Core version and runtime.
  • .NET Standard Class Library: Used for building reusable libraries that can be consumed by any .NET implementation (e.NET Framework, .NET Core, Xamarin, Unity) that supports the targeted .NET Standard version. It defines a set of APIs that all .NET implementations must provide.

9. What is CoreRT?

CoreRT was an experimental .NET runtime optimized for AOT compilation scenarios, aiming to produce self-contained, native executables with minimal dependencies. It was part of the .NET Native toolchain and has largely evolved into the AOT capabilities now integrated directly into .NET (e.g., for Blazor WebAssembly and native AOT publishing).

10. What is .NET Core SDK?

The .NET Core SDK (Software Development Kit) is a set of tools and libraries that developers use to create, build, and run .NET Core applications. It includes the .NET CLI (Command-Line Interface), the .NET runtime, libraries, and compilers.

11. What is Docker?

Docker is a platform that uses OS-level virtualization to deliver software in packages called containers. Containers are isolated, lightweight, and portable environments that bundle an application and all its dependencies, ensuring it runs consistently across different environments. .NET Core applications are often deployed in Docker containers.

12. What is Xamarin?

Xamarin is a Microsoft-owned framework for building cross-platform mobile applications (iOS, Android, UWP) using C# and .NET. It allows developers to share a significant portion of their codebase across platforms while still providing access to native APIs and UI elements.

13. How can you differentiate ASP.NET Core from .NET Core?

  • .NET Core (now .NET): This is the general-purpose, cross-platform runtime and framework for building various types of applications (console, desktop, web, cloud).
  • ASP.NET Core: This is a specific web framework built on top of .NET Core. It provides tools and libraries specifically for building modern, cloud-enabled, internet-connected applications like web apps, APIs, and real-time services.

So, ASP.NET Core is a part of the broader .NET (Core) ecosystem, focused on web development.

14. What is MSBuild in the .NET Core?

MSBuild is the build platform for Microsoft and .NET. In .NET Core, it's used to build projects by processing project files (e.g., `.csproj`). It defines an XML schema for a project file and a set of tools for building software, including compiling source code, packaging, and deployment.

15. Whether ‘debug’ and ‘trace’ are the same?

No, `Debug` and `Trace` are not the same, though they are both used for diagnostics in .NET:

  • Debug: Primarily used for debugging purposes during development. `Debug` statements are typically compiled only in debug builds and are removed in release builds to optimize performance.
  • Trace: Used for logging and tracing application behavior, often in production environments. `Trace` statements are usually compiled into both debug and release builds, allowing for monitoring and diagnostics in deployed applications.

16. What is Transfer-encoding?

Transfer-encoding is an HTTP header that indicates how the payload body of a message has been encoded to safely transfer it over HTTP. Common values include `chunked` (for streaming data) and `gzip` (for compression). It's distinct from `Content-Encoding`, which describes the encoding of the content itself.

17. Give the differences between .NET Core and Mono?

  • .NET Core (now .NET): Microsoft's official open-source, cross-platform, modular .NET implementation. Designed for modern cloud and containerized applications.
  • Mono: An older, open-source implementation of the .NET Framework, initially developed by Xamarin (now Microsoft). It aims to provide a cross-platform runtime for .NET Framework applications. While it shares similarities, it's a separate project with its own ecosystem, often used for games (Unity) and mobile apps (Xamarin before .NET MAUI).

18. Explain about types of Common Type System (CTS).

The Common Type System (CTS) defines two main categories of types:

  • Value Types: Directly contain their data. Stored on the stack (for local variables) or inline within other types. Examples: `int`, `struct`, `enum`.
  • Reference Types: Store a reference to their data, which is allocated on the heap. Examples: `class`, `string`, `array`, `delegate`.

19. What is the use of generating SQL scripts in the .NET core?

In .NET Core (especially with Entity Framework Core), generating SQL scripts is useful for:

  • Database Migrations: Creating or updating database schemas based on your EF Core model.
  • Version Control: Storing database schema changes in source control.
  • Deployment: Applying schema changes to production databases in a controlled manner.
  • Review: Allowing DBAs to review the SQL before applying changes.

20. What is the IGCToCLR interface?

The IGCToCLR interface is an internal interface within the .NET runtime, primarily used for communication between the Garbage Collector (GC) and other parts of the Common Language Runtime (CLR). It's not an interface that developers typically interact with directly, but it's crucial for the internal workings of memory management.

21. What is CoreFx?

CoreFX was the repository for the foundational class libraries for .NET Core (now simply .NET). It contained the implementation of the Base Class Library (BCL) and other core libraries that provide fundamental types, APIs, and functionalities used across all .NET applications. Its components are now part of the main `dotnet/runtime` repository.

22. What is Zero Garbage Collectors?

The term "Zero Garbage Collector" often refers to a garbage collection strategy (or lack thereof) where an application avoids allocating any managed memory on the heap, thus eliminating the need for GC pauses. This is typically achieved through careful memory management, using value types, stack allocation, or native memory directly, common in highly performance-critical or embedded systems. It's not a specific GC implementation in .NET but a design approach.

23. What is the purpose of webHostBuilder()?

In older ASP.NET Core applications (before .NET 6 minimal APIs), WebHostBuilder() was used to configure and build the web host. Its purpose was to define the server (e.g., Kestrel), configure middleware, set up the application's startup class, and integrate with logging and configuration systems. In modern .NET, much of this is simplified or handled implicitly by `WebApplication.CreateBuilder()`.

24. What is CoreCLR?

CoreCLR is the runtime component of .NET Core (now simply .NET). It's a cross-platform, open-source, and modular implementation of the Common Language Runtime (CLR). It includes the JIT compiler, garbage collector, type system, and other runtime services, enabling .NET applications to run on Windows, Linux, and macOS.

25. What are C# and F#?

  • C#: A modern, object-oriented, type-safe programming language developed by Microsoft, widely used for building a variety of applications on the .NET platform, including web, desktop, mobile, and games.
  • F#: A functional-first programming language that also supports object-oriented and imperative programming. It's part of the .NET ecosystem and is known for its conciseness, powerful type inference, and suitability for data-intensive and concurrent applications.

26. What is Dot NET Core used for?

.NET Core (now .NET) is used for building a wide range of modern applications, including:

  • Cross-platform web applications and APIs (ASP.NET Core)
  • Microservices
  • Cloud-native applications
  • Console applications
  • Desktop applications (WPF, Windows Forms, MAUI)
  • Machine learning and AI applications
  • IoT applications

27. What is .NET core?

.NET Core (now simply .NET) is a free, open-source, cross-platform managed framework for building modern, cloud-based, internet-connected applications. It's a re-imagining of the .NET Framework, designed to be modular, lightweight, and performant, supporting Windows, Linux, and macOS.
