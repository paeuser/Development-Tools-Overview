Advanced C# Features: Questions and Answers

  • Q: What is async/await in C# and why is it useful?
    A: async and await are used for asynchronous programming in C#. They allow you to write non-blocking code that can perform long-running operations (like I/O or network calls) without freezing the application. await pauses the method until the awaited task completes, but the thread is free to do other work.
  • Q: What is LINQ and what are its benefits?
    A: LINQ (Language Integrated Query) is a set of features in C# for querying collections (like arrays, lists, or databases) in a readable, declarative way. It brings SQL-like query capabilities into C# and helps write concise, type-safe, and maintainable code.
  • Q: What is a delegate in C#?
    A: A delegate is a type that represents references to methods with a specific parameter list and return type. Delegates are used for callback methods and event handling. They enable methods to be passed as parameters.
  • Q: What are events in C# and how are they related to delegates?
    A: Events are a way for a class to provide notifications to clients of that class when something happens. Events are based on delegates; they use delegates to define the signature of the event handler methods. Only the class that declares an event can invoke it, but other classes can subscribe to it.
  • Q: Can you give a simple example of using async/await? 
    public async Task DoWorkAsync() {
    await Task.Delay(1000); // Simulates work
    Console.WriteLine("Done!");
}
  • Q: Can you show a basic LINQ query? 
    var numbers = new[] { 1, 2, 3, 4 };
var evens = numbers.Where(n => n % 2 == 0).ToList();
  • Q: How do you declare and use a delegate? 
    public delegate int MathOp(int x, int y);
MathOp add = (a, b) => a + b;
int result = add(2, 3); // result is 5

SomethingHappened?.Invoke(this, EventArgs.Empty);

Popular Front-End Testing Libraries and Framework Support

  • Jest
    Jest is a widely used JavaScript testing framework for unit and integration tests. It works well with React (officially recommended), and also supports Vue and Angular (with some configuration).
    Best for: React (official), Vue, Angular
  • React Testing Library
    A library focused on testing React components in a way that resembles user interaction. It is the standard for React projects and works seamlessly with Jest.
    Best for: React only
  • Enzyme
    A React-specific testing utility for rendering and manipulating components. It is now less recommended in favor of React Testing Library.
    Best for: React only
  • Mocha
    A flexible JavaScript test framework for browser and Node.js, often used with assertion libraries like Chai. It can be used with any front-end framework.
    Best for: React, Vue, Angular, plain JS
  • Chai
    An assertion library that works with Mocha and other test runners. It is framework-agnostic.
    Best for: React, Vue, Angular, plain JS
  • Cypress
    An end-to-end (E2E) testing tool for web applications. Runs tests in the browser and is great for simulating real user behavior. Supports all major frameworks.
    Best for: React, Vue, Angular, plain JS
  • Playwright
    A modern E2E testing framework from Microsoft, supporting multiple browsers and powerful automation. Works with all major frameworks.
    Best for: React, Vue, Angular, plain JS
  • Jasmine
    A behavior-driven development (BDD) framework for testing JavaScript code, often used for Angular projects.
    Best for: Angular (official), React, Vue
  • Karma
    A test runner that launches browsers and runs tests, often used with Jasmine or Mocha in Angular projects.
    Best for: Angular (official), React, Vue
