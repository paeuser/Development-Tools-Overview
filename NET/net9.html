.NET 9

Release Date: November 2024

New Features

  • Native AOT Improvements: Faster startup time and smaller executables, ideal for microservices and containerized apps.
  • DATAS Garbage Collector: Smarter memory management that adapts heap sizes dynamically.
  • Control-flow Enforcement Technology (CET): Default on Windows for improved runtime security.
  • Cloud-native Optimizations: Smaller container images, better support for Kubernetes and Dapr, and new tools for distributed systems.
  • .NET Aspire (Preview): A new stack of NuGet packages to simplify building observable, scalable cloud apps.
  • Enhanced System.Text.Json: Better serialization options and performance.
  • LINQ Enhancements: New methods like CountBy, AggregateBy, and WhereNotNull.
  • NuGet Vulnerability Audits: Automatic scanning of transitive package vulnerabilities during build.
  • Improved MSBuild: Parallel test execution and workload version pinning.
  • PersistentAssemblyBuilder API: Emit and debug dynamic assemblies more easily.
  • PriorityQueue Enhancements: Ability to remove specific items by value.
  • ARM SVE/SVE2 Support: Vectorized hardware acceleration for ARM64 devices.
  • Windows Forms Updates: Support for async dialogs, experimental dark mode, and visual enhancements.
  • .NET MAUI Improvements: Better mobile/web UI integration with new controls and performance tweaks.
  • Cryptography Updates: KMAC support and one-shot hashing APIs in CryptographicOperations.

C# 13 Highlights

  • params Span<T>: Enables efficient parameter passing for performance-critical APIs.
  • New Lock Object Support: Safer and more expressive thread synchronization options.
  • Escape Sequence \e: Represents the ASCII escape character in strings.
  • Implicit Indexers & Method Group Enhancements: Improved overload resolution and indexing from end of arrays.

.NET SDK

The .NET SDK (Software Development Kit) is a set of libraries, tools, and utilities for building, running, and publishing .NET applications. It includes:

  • Compilers for C#, F#, and Visual Basic.
  • The dotnet CLI (dotnet command-line tool) for creating, building, running, testing, and publishing projects.
  • Core libraries and runtime components needed to develop and execute .NET apps.
  • Project templates for quickly starting new applications, libraries, and services.
  • Tools for dependency management, package restore, and integration with NuGet (the .NET package manager).
  • Workload version pinning: Keep consistent SDK behavior across team environments.

The .NET SDK is cross-platform and can be installed on Windows, macOS, and Linux. It is updated regularly with new features, performance improvements, and support for the latest .NET versions (such as .NET 9).

To check your installed SDKs, run dotnet --list-sdks in your terminal. To create a new project, use dotnet new followed by the template name.
