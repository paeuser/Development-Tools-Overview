Assurity ECS Angular App – Frontend

Introduction for Business Users
The Assurity ECS Angular App is the main web interface for the Assurity platform. It allows business users to easily view, manage, and interact with products, customers, and business processes from any device with a web browser. The app is designed to be intuitive, responsive, and secure, making it simple for users to get their work done efficiently.

What version of Angular was used?
The application is built with Angular version 14 (see package.json for details).
What test framework was used?
The app uses Karma and Jasmine for automated testing, which are standard for Angular projects.
How does this app call the API and external sites?
The Angular app communicates with the Assurity ECS API using secure HTTP (REST) requests. It sends and receives data such as product information, user details, and business actions. For authentication, it uses secure tokens (JWT) provided by the backend. The app can also connect to external services (like e-signature or email providers) through the backend API, ensuring all integrations are managed securely and centrally.

Related: See the Assurity ECS API (Backend) for details on the backend services that power this Angular frontend.

What is Bootstrap?

Bootstrap is a popular open-source CSS framework for building responsive, mobile-first web applications quickly and easily. It provides a collection of ready-to-use CSS classes and JavaScript components for layout, typography, forms, buttons, navigation, and more. By using Bootstrap, developers can ensure their applications look modern and work well on all devices without writing a lot of custom CSS.

  • Responsive Design: Automatically adapts layouts for desktops, tablets, and mobile devices.
  • Prebuilt Components: Includes cards, modals, navbars, alerts, and more for rapid UI development.
  • Consistent Styling: Ensures a professional and uniform look across the application.
  • Customizable: You can override Bootstrap styles with your own CSS or use its utility classes for quick adjustments.

In this project, Bootstrap is used to create a clean, user-friendly interface and to make sure the app works well on both desktop and mobile devices. For example, the container, card, mb-3, and shadow classes in this page are all from Bootstrap.

Technical Details
  • Framework: Angular 14, using TypeScript for type safety and maintainability.
  • Architecture: Follows a modular structure with feature modules, shared modules, and core modules for separation of concerns and scalability.
  • State Management: Uses Angular services and RxJS for managing state and asynchronous data streams.
  • API Communication: Utilizes Angular's HttpClient for RESTful API calls, with JWT token-based authentication for secure access.
  • Routing: Implements Angular Router for navigation and lazy loading of modules to optimize performance.
  • Forms: Uses Reactive Forms for robust form validation and dynamic form controls.
  • Testing: Automated unit and integration tests are written using Jasmine and Karma; code coverage is tracked for quality assurance.
  • Styling: Employs Bootstrap for responsive design and SCSS for custom styles.
  • Security: Implements route guards, input validation, and secure token storage to protect user data and application integrity.
  • Deployment: Built and deployed using Angular CLI, with CI/CD pipelines (e.g., Azure DevOps) for automated testing and deployment.
  • Integration: Connects to the white-labeled Assurity ECS backend API and external services (e.g., e-signature, email) through secure endpoints.

Common Interview Questions You Can Answer from This Project:

  • How do you structure a large Angular application for maintainability and scalability?
    The project uses a modular architecture, separating features into dedicated modules (feature, shared, and core modules). This approach allows for code reusability, easier testing, and scalable development as the app grows.
  • How do you handle authentication and secure API communication in Angular?
    Authentication is managed using JWT tokens. The Angular app stores the token securely and attaches it to HTTP requests via interceptors. Route guards protect sensitive routes, and all API calls are made over HTTPS.
  • What strategies do you use for state management and asynchronous data in Angular?
    Angular services and RxJS observables are used for state management and handling asynchronous data. This enables reactive programming, real-time updates, and efficient data flow between components.
  • How do you implement and test forms with complex validation?
    Reactive Forms are used for building forms with dynamic and custom validation logic. Unit tests are written for form controls and validation using Jasmine and Karma.
  • How do you ensure code quality and reliability in an Angular project?
    The project uses automated unit and integration tests (Jasmine/Karma), code reviews, and code coverage tools. Linting and formatting are enforced for consistency.
  • How do you integrate Angular with backend APIs and external services?
    Angular's HttpClient is used to communicate with the backend API and external services. All integrations are handled through secure, centralized endpoints, and error handling is implemented for reliability.
  • What are best practices for deploying and maintaining Angular applications in production?
    The app is built and deployed using Angular CLI, with CI/CD pipelines (Azure DevOps) for automated testing and deployment. Best practices include environment-based configuration, code splitting, lazy loading, and monitoring.
