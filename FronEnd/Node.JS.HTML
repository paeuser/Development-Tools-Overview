Node.js Interview Questions & Answers

1. What is Node.js and why is it popular?
Node.js is a runtime environment that allows you to run JavaScript on the server side. It's popular because it's fast, event-driven, non-blocking, and enables full-stack JavaScript development.
2. Explain the event loop in Node.js.
The event loop is the core mechanism that handles asynchronous operations in Node.js. It allows Node.js to perform non-blocking I/O by offloading operations and executing callbacks when events are ready.
3. What is the difference between process.nextTick(), setImmediate(), and setTimeout()?
  • process.nextTick(): Executes a callback after the current operation completes, before the event loop continues.
  • setImmediate(): Executes a callback on the next iteration of the event loop.
  • setTimeout(): Schedules a callback after a minimum delay.
4. How do you handle uncaught exceptions in Node.js?
You can use process.on('uncaughtException', handler) to catch uncaught exceptions, but it's better to handle errors at the source and use promises or callbacks with error handling.
5. What is middleware in Express.js?
Middleware are functions that have access to the request and response objects in Express.js. They can modify requests, responses, or end the request-response cycle. Examples: logging, authentication, error handling.
6. How do you manage packages in Node.js?
Node.js uses npm (Node Package Manager) or yarn to manage packages. You can install, update, and remove packages using commands like npm install, npm update, and npm uninstall.
7. What is a callback hell and how can you avoid it?
Callback hell is deeply nested callbacks that make code hard to read and maintain. You can avoid it by using promises, async/await, or modularizing your code.
8. How do you create a simple HTTP server in Node.js?
const http = require('http');
http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello World');
}).listen(3000);
9. What is the difference between require and import in Node.js?
require is CommonJS syntax, used in most Node.js projects. import is ES Module syntax, supported in newer Node.js versions with type: "module" in package.json.
10. How do you handle environment variables in Node.js?
You can use process.env to access environment variables. Tools like dotenv can load variables from a .env file.

Node Package Management (npm)

npm (Node Package Manager) is the default package manager for Node.js. It allows developers to install, share, and manage third-party libraries (called packages or modules) for their Node.js projects.

  • Installing Packages: Use npm install package-name to add a package to your project.
  • package.json: This file lists your project's dependencies and scripts. It is created with npm init.
  • Global vs Local: Packages can be installed globally (available everywhere) or locally (only in your project).
  • Updating Packages: Use npm update to update dependencies.
  • Removing Packages: Use npm uninstall package-name to remove a package.
  • Running Scripts: Define custom scripts in package.json and run them with npm run script-name.

npm makes it easy to manage project dependencies and automate common development tasks in Node.js applications.
