How We Created This Vite + React App

Step 1: Scaffolding the App
We started by creating a new React app using Vite, a fast build tool for modern web projects. In your terminal, we ran:
npx create-vite@latest . --template react-ts
This set up a new project with React and TypeScript.

Step 2: Installing Dependencies
After scaffolding, we installed all the required packages by running:
npm install

Setting Up and Running Our First Test

Step 3: Adding Testing Tools
To test our React components, we installed Vitest (a fast test runner), @testing-library/react (for testing React UIs), and @testing-library/jest-dom (for better assertions):
npm install --save-dev vitest @testing-library/react @testing-library/jest-dom @testing-library/user-event jsdom

Step 4: Configuring Vitest
We updated vite.config.ts to use the jsdom environment and set up a test setup file. This makes sure our tests run in a browser-like environment and have access to helpful matchers.

Step 5: Writing Our First Test
We created src/App.test.tsx to check that our main page shows the heading Vite + React.
Example test code:

Step 6: Running the Test
We ran our test suite with:

Example Test Code

This is the code for our first test. It checks that the heading Vite + React is displayed on the page. Each line is commented to help you understand what it does:

// Import render and screen utilities for testing React components
import { render, screen } from '@testing-library/react';
// Import Vitest's test functions and assertion library
import { describe, it, expect } from 'vitest';
// Import the App component to be tested
import App from './App';

// Group related tests for the App component
describe('App', () => {
  // Test that the heading 'Vite + React' is rendered
  it('renders the Vite + React heading', () => {
    // Render the App component in a virtual DOM
    render();
    // Assert that the heading is present in the document
    expect(screen.getByText(/Vite \\+ React/i)).toBeInTheDocument();
  });
});

What is this test doing?

  • Rendering: It creates a virtual version of your App component, just like it would appear in the browser.
  • Querying: It looks for the text Vite + React in the rendered output.
  • Asserting: It checks that this text is actually present on the page. If it is, the test passes!
