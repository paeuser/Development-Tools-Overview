React Interview Questions & Answers

1. What is JSX?
JSX stands for JavaScript XML. It allows us to write HTML inside JavaScript and place them in the DOM without using functions like appendChild() or createElement(). It provides syntactic sugar for React.createElement().
2. What are the differences between functional and class components?
Functional components are JavaScript functions, while class components use ES6 classes. Functional components use hooks for state and lifecycle, while class components use this.state and lifecycle methods. Hooks make functional components as powerful as class components.
3. What is the virtual DOM? How does React use it to render the UI?
The virtual DOM is a lightweight copy of the real DOM. React updates the virtual DOM, compares it to the previous version, and only updates the real DOM where changes occurred, making UI updates efficient.
4. What are the differences between controlled and uncontrolled components?
Controlled components have their input values managed by React state, while uncontrolled components rely on the DOM for their state. Controlled components use onChange and state, while uncontrolled use refs.
5. What are props in React?
Props are inputs to a React component, passed as attributes. They are read-only and allow data to flow from parent to child components.
6. Explain React state and props.
State is local, mutable data managed within a component. Props are immutable and passed from parent to child. State changes trigger re-renders; props are read-only.
7. Explain about types of side effects in React component.
Effects without cleanup (e.g., data fetching) and effects with cleanup (e.g., subscriptions) are managed with useEffect. Cleanup prevents memory leaks.
8. What is prop drilling in React?
Prop drilling is passing data from a parent to deeply nested child components through intermediate components, which can make code harder to maintain.
9. What are error boundaries?
Error boundaries are React components that catch JavaScript errors in their child component tree and display a fallback UI instead of crashing the app. Implemented using getDerivedStateFromError and componentDidCatch.
10. What are React Hooks?
Hooks are built-in functions (like useState, useEffect) that let you use state and lifecycle features in functional components.
11. What are the rules for using React Hooks?
Hooks must be called at the top level of a React function component or custom hook, not inside loops, conditions, or nested functions.
12. What is the use of useEffect React Hook?
useEffect lets you perform side effects in functional components, such as data fetching, subscriptions, or manual DOM changes. It runs after render and can clean up effects.
13. Why do React Hooks make use of refs?
Refs let you access and interact with DOM elements directly. useRef is used in function components for focus, media playback, or integrating with third-party libraries.
14. What are Custom Hooks?
Custom Hooks are JavaScript functions that start with use and call other hooks. They allow you to reuse stateful logic across components.
15. What are the lifecycle methods of React?
Lifecycle methods are special methods in class components: constructor, render, componentDidMount, shouldComponentUpdate, componentDidUpdate, componentWillUnmount, etc. Hooks provide similar functionality in functional components.
16. What are Higher Order Components?
A Higher-Order Component (HOC) is a function that takes a component and returns a new component, often used for code reuse and logic abstraction.
