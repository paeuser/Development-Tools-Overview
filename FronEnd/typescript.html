TypeScript Questions and Answers

What is TypeScript?

TypeScript is a strongly typed programming language that builds on JavaScript, giving you better tooling at any scale.

What are the benefits of using TypeScript?

  • Static typing helps catch errors early during development.
  • Improved code readability and maintainability.
  • Enhanced IDE support with autocompletion and refactoring tools.
  • Compatibility with existing JavaScript code.

How do you install TypeScript?

You can install TypeScript globally using npm with the following command:

npm install -g typescript

What is a .ts file?

A .ts file is a TypeScript file that contains TypeScript code. It is transpiled into JavaScript for execution in browsers or Node.js.

How do you compile TypeScript code?

Use the TypeScript compiler (tsc) to compile .ts files into .js files. For example:

tsc filename.ts

What is the difference between TypeScript and JavaScript?

TypeScript is a superset of JavaScript that adds static typing and other features, while JavaScript is a dynamic, loosely typed language.

TypeScript Interview Questions and Answers

What are TypeScript's key features?

TypeScript offers static typing, interfaces, classes, and modules, which enhance JavaScript development.

What is the difference between an interface and a type in TypeScript?

Both are used for defining object shapes, but interfaces can be extended or implemented, while types are more flexible and can represent unions or intersections.

What is the purpose of TypeScript's "any" type?

The "any" type allows you to opt out of type checking for a variable, making it compatible with any type.

How does TypeScript handle null and undefined?

TypeScript has strict null checks enabled by default, requiring explicit handling of null and undefined values.

What is the difference between "unknown" and "any" in TypeScript?

The "unknown" type is safer than "any" because you must perform type checks before using it, while "any" bypasses type checking entirely.

How do you define a function in TypeScript?

You define a function with type annotations for its parameters and return value. For example:

function add(a: number, b: number): number {
    return a + b;
}

What is TypeScript's "readonly" modifier?

The "readonly" modifier ensures that a property cannot be reassigned after its initial assignment.

TypeScript Utility Types

What are Utility Types?
TypeScript provides several built-in utility types that help you transform, compose, and work with types more efficiently. These utilities make it easier to express common type transformations and constraints, improving code safety and readability.

Utility types are generic types provided by TypeScript, such as Partial, Pick, Omit, Record, and Readonly. They are especially useful for manipulating existing types or creating new types based on existing structures.

Common Utility Types and Examples

// Partial: Makes all properties optional
interface User { name: string; age: number; }
type PartialUser = Partial; // { name?: string; age?: number; }

// Pick: Selects a subset of properties
type NameOnly = Pick; // { name: string; }

// Omit: Removes specified properties
type WithoutAge = Omit; // { name: string; }

// Record: Creates an object type with specific keys and value type
type Example = Record<"a" | "b", number>; // { a: number; b: number; }

// Readonly: Makes all properties readonly
type ReadonlyUser = Readonly; // { readonly name: string; readonly age: number; }

Explanation: Record<K, T>

Record<K, T> constructs an object type with keys K and values of type T.
Example: Record<"a" | "b", number> is equivalent to { a: number; b: number; }.

Simple Sample: 

const scores: Record<"math" | "english", number> = { math: 95, english: 88 };
Here, scores must have both math and english properties, each with a number value.
